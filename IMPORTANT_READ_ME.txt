================================================================================
          重要！！！ 必须手动执行的安装步骤
================================================================================

你的后端仍然找不到sqlalchemy，因为启动脚本没有使用虚拟环境。

【关键问题】
旧的启动脚本使用的是系统Python，不是虚拟环境Python。
我已经修改了启动脚本，但你需要重新安装依赖到虚拟环境中。

================================================================================
          【步骤1】打开CMD窗口
================================================================================

1. 按 Win+R 键
2. 输入: cmd
3. 按回车

================================================================================
          【步骤2】复制并执行以下命令（逐行执行）
================================================================================

第1行 - 进入虚拟环境目录：
cd /d "C:\Users\hxm\有用的东西\ai-solver-mvp\backend\venv\Scripts"

第2行 - 安装sqlalchemy：
pip.exe install sqlalchemy

第3行 - 安装passlib：
pip.exe install passlib[bcrypt]

第4行 - 安装python-jose：
pip.exe install python-jose[cryptography]

第5行 - 安装python-multipart：
pip.exe install python-multipart

第6行 - 安装reportlab：
pip.exe install reportlab

第7行 - 安装python-docx：
pip.exe install python-docx

第8行 - 验证安装：
pip.exe list | findstr "sqlalchemy passlib jose multipart"

如果第8行显示了这些包，说明安装成功！

================================================================================
          【步骤3】重新启动后端
================================================================================

方法1（推荐）：
1. 关闭当前运行后端的窗口
2. 双击运行：启动服务.bat

方法2（手动）：
1. 打开新的CMD窗口
2. 执行：
   cd /d "C:\Users\hxm\有用的东西\ai-solver-mvp\backend"
   venv\Scripts\activate.bat
   python -m uvicorn main:app --reload

3. 看到启动日志应该显示：
   ✅ 数据库和认证模块导入成功
   ✅ 用户认证路由已加载

================================================================================
          【步骤4】测试注册
================================================================================

访问：http://localhost:5173/?mode=new
尝试注册新用户

================================================================================
          为什么会出现这个问题？
================================================================================

1. 你的Python环境有两个：
   - 全局Python（C:\Users\hxm\AppData\Local\Programs\Python\Python313）
   - 虚拟环境Python（项目的backend\venv\Scripts\python.exe）

2. 旧的启动脚本使用的是全局Python
   我们安装的依赖也装到了全局Python

3. 但后端实际需要在虚拟环境中运行
   所以找不到依赖

4. 解决方案：
   - 在虚拟环境中安装依赖（步骤2）
   - 使用虚拟环境启动后端（步骤3）
